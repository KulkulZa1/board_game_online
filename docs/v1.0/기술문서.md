## 1. 프로젝트 개요

아는 사람끼리 카카오톡 링크로 대국하는 **1대1 온라인 보드게임 플랫폼**.
Render.com 클라우드 서버로 고정 URL 운영. 별도 회원가입 없이 링크 공유만으로 즉시 플레이 가능.

### 지원 게임

| 게임 | 규칙 특이사항 |
|------|-------------|
| 체스 | chess.js 서버 검증, 폰 승급, 캐슬링, 앙파상 |
| 오목 | 렌주 룰: 정확히 5개 연결 승리, 장목(6+)은 무효 |
| 사목 | 7×6 중력 낙하. 호스트=빨강(선공), 게스트=노랑 |
| 오셀로 | 유효 수 없으면 자동 패스, 양측 모두 없으면 종료 |
| 인디언 포커 | waiting→deal→bet→showdown 상태 기계, 칩 소진 시 종료 |
| 체커 | 강제 점프 룰, 연속 점프(mustJump), 킹 승격 |

### 설계 원칙

- **서버 권위**: 모든 수의 유효성은 서버에서 검증. 클라이언트는 UI 렌더링만 담당
- **재접속 지원**: UUID 토큰 기반으로 새로고침/연결 끊김 후 10분 이내 복귀 가능
- **외부 라이브러리 최소화**: 사운드는 Web Audio API, Rate Limit은 in-memory 구현
- **클라우드 친화**: Render.com 환경에서 `process.env.PORT` 사용, CORS `*` 허용

## 4. 서버 설계 (server.js)

### 4-1. 상태 저장

```
rooms    : Map<roomId, RoomState>   — 모든 방 상태 (in-memory)
tokenMap : Map<playerToken, { roomId, role }>  — 재접속 토큰 역방향 조회
```

서버 재시작 시 모든 상태 소멸. DB 없음.

### 4-2. 타이머 설계

```
setInterval(500ms) → 모든 active 방 순회
  → lastTickAt === null  : skip (플레이어 끊김 → 일시 정지)
  → timers.white === null: skip (무제한 모드)
  → elapsed = now - lastTickAt
  → 현재 턴 플레이어 시간 차감
  → timer:tick 이벤트 브로드캐스트
  → 시간 ≤ 0 → endGame(timeout)
  → 인디언 포커 타임아웃 → 자동 call (기권 아님)
```

**핵심**: `lastTickAt = null` 상태에서 `Date.now() - null = ~1.7조ms` 가 되어
즉시 timeout이 발생하는 버그를 `if (lastTickAt === null) continue` 로 방지.

### 4-3. 방 생명주기

```
대기(waiting) ──[게스트 참가]──▶ 진행(active) ──[게임 종료]──▶ 완료(finished)
     │                                │
  30분 후 자동 삭제             끊김 후 10분 내 미복귀 → disconnect 패배
```

### 4-4. 게임 모듈 패턴

각 게임은 `handleXxxMove(socket, room, role, data)` 함수로 분리:

```
game:move 이벤트 → room.gameType 분기
  chess    → handleChessMove()
  omok     → handleOmokMove()
  connect4 → handleConnect4Move()
  othello  → handleOthelloMove()
  checkers → handleCheckersMove()
```

인디언 포커만 별도 이벤트 (`indianpoker:action`) 사용.

### 4-5. 인디언 포커 상태 기계

```
waiting
  └─[게스트 참가 + 500ms 후]─▶ deal
      ├─ 양측에게 상대 카드만 발송 (보안: io.to(socketId).emit)
      └─▶ bet (betTurn = 'guest' 먼저)
            ├─ fold  → showdown (폴드 플레이어 패배)
            ├─ raise → 레이즈 최대 3회 제한 → betTurn 전환
            └─ call  → guest call: 호스트 차례 → host call: showdown
  └─▶ showdown
        ├─ 높은 숫자 승리 (동점: 호스트 승)
        ├─ 칩 정산
        └─ chips > ante: 4초 후 다음 라운드 (deal로 복귀)
           chips = 0:   game:over (chips-depleted)
```

### 4-6. 체커 mustJump 연속 점프

```
handleCheckersMove() 처리 후:
  점프 성공 + 추가 점프 가능
    → mustJump = { row, col } (현재 말 위치)
    → 턴 전환 없음 (같은 플레이어 계속)
    → nextValidMoves = getValidCheckersMovesForPiece(말, 점프만)
  추가 점프 불가
    → mustJump = null
    → 턴 전환
    → nextValidMoves = getAllCheckersValidMoves(다음 플레이어)
```

### 4-7. 오셀로 패스 처리

```
handleOthelloMove() 처리 후:
  상대 유효 수 계산
    ├─ 있음: 턴 전환 (정상)
    └─ 없음: 패스 (pass=true 포함 broadcast)
        현재 플레이어 유효 수 계산
          ├─ 있음: 현재 플레이어 계속
          └─ 없음: 양측 모두 없음 → 게임 종료 (돌 수 비교)
```

### 4-8. graceful shutdown

```
SIGTERM / SIGINT / POST /admin/shutdown
  → io.emit('server:shutdown')  — 모든 클라이언트에 알림
  → server.close()              — 새 연결 거부
  → .shutdown-key 파일 삭제
  → process.exit(0)
  // 5초 내 미완료 시 강제 종료
```

## 5. 클라이언트 설계

### 5-1. 멀티 게임 보드 전환 (game.js)

```js
// IIFE 모듈 패턴으로 각 게임 보드 격리
window.Connect4Board = (function() { ... })();
window.OthelloBoard  = (function() { ... })();
window.IndianPoker   = (function() { ... })();
window.CheckersBoard = (function() { ... })();

// ActiveBoard 포인터로 현재 게임 보드 참조
let ActiveBoard = null;

function switchBoardArea(type) {
  ['chess','omok','connect4','othello','indianpoker','checkers'].forEach(t => {
    document.getElementById(`${t}-board-area`).style.display = t === type ? '' : 'none';
  });
}
```

각 보드 모듈 공통 API: `init()`, `setMyTurn()`, `updateAfterMove()`, `clearSelection()`

### 5-2. board.js — 체스판

**렌더링**: 64개 `div.cell` 전체 재렌더링 방식.

**클릭 흐름**:
```
첫 클릭  → 내 기물 선택 → chess.moves({ square }) 로 합법 수 계산
두 번째  → 합법 수 칸이면 attemptMove()
          → 승급 여부 확인 → showPromotion() or commitMove()
          → onMoveCallback({ from, to, promotion }) → socket.emit('game:move')
```

### 5-3. timer.js — 타이머 보간

서버 500ms tick 사이를 `requestAnimationFrame` 으로 보간하여 부드럽게 표시합니다.

```
timer:tick 수신 → _white/_black 저장 + _syncedAt = performance.now()
rAF 루프: elapsed = performance.now() - _syncedAt
          display = max(0, activeColor_ms - elapsed)
```

### 5-4. 인디언 포커 UI (indian-poker.js)

상대 카드는 서버에서 `socket.emit()`(개인 발송)으로 수신. 내 카드는 "?" 표시.
배팅 액션: 인라인 onclick `window._ipAction('call'|'raise'|'fold')` → 1.5초 debounce → `socket.emit('indianpoker:action')`.

### 5-5. sound.js — 사운드 엔진

Web Audio API, 외부 파일 없음. 첫 사용자 인터랙션 시 lazy AudioContext 생성.

### 5-6. admin.html — 관리자 대시보드

`/api/status` 를 3초마다 폴링하여 실시간 서버 현황을 표시.

## 6. Socket.io 이벤트 명세

### 6-1. Client → Server

| 이벤트 | 페이로드 | 설명 |
|--------|---------|------|
| `room:create` | `{ gameType, hostColor, timeControl }` | 방 생성 |
| `room:join` | `{ roomId }` | 게스트 참가 |
| `room:reconnect` | `{ playerToken }` | 재접속 |
| `room:spectate` | `{ roomId }` | 관전 요청 |
| `game:move` | 게임별 상이 (하단 참고) | 수 두기 |
| `game:resign` | — | 기권 (서버: 분당 3회 제한) |
| `game:draw:offer` | — | 무승부 제안 체스 전용 (서버: 분당 5회 제한) |
| `game:draw:respond` | `{ accept: bool }` | 무승부 응답 |
| `game:rematch:request` | — | 재대국 요청 |
| `game:rematch:respond` | `{ accept: bool }` | 재대국 응답 |
| `indianpoker:action` | `{ action: 'call'|'raise'|'fold' }` | 인디언 포커 배팅 |
| `chat:send` | `{ text }` | 채팅 전송 |

#### game:move 페이로드 (게임별)

| 게임 | 페이로드 |
|------|---------|
| 체스 | `{ from, to, promotion? }` |
| 오목 | `{ row, col }` |
| 사목 | `{ col }` |
| 오셀로 | `{ row, col }` |
| 체커 | `{ from: {row,col}, to: {row,col}, jumps: [{row,col},...] }` |

### 6-2. Server → Client (공통)

| 이벤트 | 페이로드 | 설명 |
|--------|---------|------|
| `room:created` | `{ roomId, playerToken, hostColor, gameType }` | 방 생성 완료 |
| `room:joined` | `{ playerToken, guestColor, roomId, gameType }` | 참가 완료 |
| `room:guest:joined` | `{ guestColor }` | 호스트에게 게스트 입장 알림 |
| `room:error` | `{ code, message }` | 오류 |
| `game:start` | `{ board, currentTurn, timers, gameType, ... }` | 게임 시작 |
| `game:state` | *(전체 상태, 재접속 동기화)* | 재접속 시 전체 상태 |
| `game:move:made` | 게임별 상이 (하단 참고) | 수 결과 |
| `game:move:invalid` | `{ reason }` | 유효하지 않은 수 |
| `game:over` | `{ winner, reason, moves }` | 게임 종료 |
| `timer:tick` | `{ white, black, activeColor }` | 500ms 타이머 |
| `player:disconnected` | `{ role }` | 상대방 끊김 |
| `player:reconnected` | `{ role }` | 상대방 재접속 |
| `game:draw:offered` | — | 무승부 제안 수신 |
| `game:draw:declined` | — | 무승부 거절 수신 |
| `rematch:requested` | — | 재대국 요청 수신 |
| `rematch:declined` | — | 재대국 거절 수신 |
| `rematch:accepted` | — | 재대국 수락 |
| `chat:message` | `{ role, text, ts }` | 채팅 |
| `server:shutdown` | — | 서버 종료 알림 |

#### game:move:made 페이로드 (게임별)

| 게임 | 추가 필드 |
|------|---------|
| 체스 | `fen, pgn, check` |
| 오목 | `board, move: {row,col,color}` |
| 사목 | `board, move: {row,col,color}, colHeights, winCells?` |
| 오셀로 | `board, move: {row,col,color,flipped}, validMoves, scores, pass?` |
| 체커 | `board, move: {...}, validMoves, mustJump, captured, promoted` |

### 6-3. Server → Client (인디언 포커 전용)

| 이벤트 | 페이로드 | 설명 |
|--------|---------|------|
| `indianpoker:dealt` | `{ opponentCard, pot, chips, ante }` | 카드 딜 (개인 발송) |
| `indianpoker:bet:turn` | `{ betTurn, pot, chips, lastAction }` | 배팅 차례 |
| `indianpoker:showdown` | `{ hostCard, guestCard, winner, reason, pot, chips }` | 쇼다운 결과 |

### 6-4. Server → Client (관전자 전용)

| 이벤트 | 페이로드 | 설명 |
|--------|---------|------|
| `spectator:approved` | `{ gameType, board, ...전체 상태 }` | 관전 승인 + 상태 |
| `spectator:hint` | 게임별 상이 | 힌트 정보 |

## 7. REST API 명세

| 메서드 | 경로 | 인증 | 설명 |
|--------|------|------|------|
| `GET` | `/api/status` | 없음 | 서버 현황. localhost 접속 시 `shutdownKey` 포함 |
| `POST` | `/admin/shutdown` | body `{ key }` | 서버 종료 |

### GET /api/status 응답 예시

```json
{
  "uptime": 3742,
  "rooms": { "active": 2, "waiting": 1, "total": 3 },
  "players": { "connected": 4 },
  "shutdownKey": "uuid-only-on-localhost"
}
```

`shutdownKey` 는 `req.hostname === 'localhost'` 일 때만 포함됩니다.

## 8. 데이터 구조

### RoomState (공통)

```js
{
  id, hostToken, guestToken,
  gameType: 'chess'|'omok'|'connect4'|'othello'|'indianpoker'|'checkers',
  hostColor, guestColor,
  timeControl: { minutes: number | null },
  status: 'waiting' | 'active' | 'finished',
  currentTurn: 'white' | 'black' | 'red' | 'yellow',
  moves: [],
  timers: { white, black, activeColor, lastTickAt },
  players: {
    host:  { socketId, connected },
    guest: { socketId, connected }
  },
  chat: [{ role, text, ts }],
  winner, rematchRequest, cleanupTimer
}
```

### RoomState 게임별 추가 필드

| 게임 | 추가 필드 |
|------|---------|
| 체스 | `chess` (Chess 인스턴스), `fen`, `pgn` |
| 오목·사목·오셀로·체커 | `board` (2D 배열) |
| 사목 | `colHeights` (길이 7 배열, 각 열 쌓인 수) |

| 체커 | `mustJump` (`{row,col}` or `null`) |

## 9. 게임별 핵심 흐름

### 공통 방 생성/참가

```
호스트: room:create { gameType, ... }
  → server: rooms.set(roomId, createRoomState())
  → room:created { roomId, playerToken }
  → game.html?room=<id>&game=<type>

게스트: 링크 클릭 → room:join { roomId }
  → startGame() 호출 → 양쪽: game:start 수신
```

### 수 두기 (체스 예시)

```
클라이언트: 클릭 → socket.emit('game:move', { from, to })
서버: 턴 검증 → chess.move() → FEN/PGN 업데이트
  → io.to(roomId).emit('game:move:made')
  → 체크메이트/무승부 감지 → endGame()
클라이언트: game:move:made → Board.updateAfterMove() → 사운드
```

### 인디언 포커 라운드

```
startIndianPokerRound():
  1. chips -= ante (양측)
  2. deck에서 각자 1장 → 상대에게만 발송
  3. betTurn = 'guest'
  4. indianpoker:bet:turn broadcast

게스트 배팅:
  fold → 호스트 승, 새 라운드
  raise → 레이즈(최대 3회), 호스트 차례
  call → 호스트 차례 (betTurn = 'host')

호스트 배팅:
  fold/raise/call 후 → doShowdown()

doShowdown():
  hCard.rank vs gCard.rank → 높은 쪽 pot 획득
  chips 정산 → 0 이하면 game:over, 아니면 새 라운드
```

### 체커 연속 점프

```
handleCheckersMove():
  점프 성공 + getJumpCheckersMovesForPiece(to) 존재
    → room.mustJump = to
    → 턴 유지, 해당 말만 이동 가능
    → validMoves = getValidCheckersMovesForPiece(to)
  점프 없거나 추가 점프 불가
    → room.mustJump = null
    → 턴 전환
```

## 10. 보안 설계

| 항목 | 내용 |
|------|------|
| Rate limit (방 생성) | create 5/60s |
| Rate limit (입장) | join 10/60s, reconnect 5/60s |
| Rate limit (게임) | move 30/10s, chat 20/10s, ipaction 10/30s |
| Rate limit (기권) | resign 3/60s |
| Rate limit (무승부) | drawOffer 5/60s |
| 클라이언트 버튼 보호 | 무승부: 5초 딜레이·3회→60초 비활성 / 기권: 3초 쿨다운·이중 emit 방지 |
| 인디언 포커 액션 | fold/call/raise 1.5초 debounce |
| 입력 검증 | gameType 허용 목록, 좌표 범위, chat 텍스트 길이 |
| 셧다운 키 | UUID, `.shutdown-key` 파일 (mode 0o600), POST body 전달 |
| 방 개수 제한 | 동시 최대 20개 |
| /api/status | shutdownKey 는 localhost 접속 시에만 응답에 포함 |
| 인디언 포커 카드 | 개인 `socket.emit()` 으로만 발송, `io.to(room).emit()` 절대 사용 금지 |

## 11. 배포 구조

### 11-1. 운영 서버 (Render.com — 기본)

```
인터넷 사용자
     │ HTTPS + WebSocket
     ▼
Render.com CDN
     │
Node.js 서버 (board-game-online.onrender.com)
  PORT: process.env.PORT (Render 자동 주입, 기본 10000)
  CORS: origin: '*'
```

- 고정 URL: `https://board-game-online.onrender.com`
- 자동 배포: GitHub `main` 브랜치 push 시 트리거
- 슬립 방지: UptimeRobot 14분 핑 (`/api/status`)
- `render.yaml` 로 빌드·시작 명령·환경 변수 관리

### 11-2. 로컬 서버 (Cloudflare Tunnel — 선택)

```
인터넷 사용자
     │ HTTPS + WebSocket
     ▼
Cloudflare CDN (글로벌)
     │
cloudflared.exe (로컬 PC)
     │ HTTP + WS
     ▼
Node.js 서버 (localhost:3000)
```

- URL이 매번 변경됨 (임시 터널)
- PC가 켜져 있어야 접속 가능
- 개발/테스트용으로 사용
