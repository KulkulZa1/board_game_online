## 미해결 항목 상세

---

### 1. 서버 재시작 시 진행 중 게임 소멸

**심각도**: 상 | **가능성**: 낮음 (Render 자동 재시작은 드물게 발생)

모든 게임 상태가 서버 메모리(in-memory Map)에만 존재합니다.
서버가 재시작되면 진행 중인 모든 게임 데이터가 사라집니다.

**현재 완화책**

- Render.com이 SIGTERM을 보낸 후 graceful shutdown 실행
- `server:shutdown` 이벤트로 클라이언트에 종료 알림
- 재시작 후 새 방에서 재시작 가능

**권장 해결책** (필요 시)

```js
// gracefulShutdown() 내에 추가
const dump = {};
for (const [id, room] of rooms) {
  if (room.status === 'active') {
    dump[id] = { fen: room.fen, moves: room.moves, timers: room.timers };
  }
}
fs.writeFileSync('state-dump.json', JSON.stringify(dump));
```

---

### 2. Render.com 무료 플랜 슬립 (15분)

**심각도**: 중 | **가능성**: 높음 (사용자 없을 때 항상 발생)

Render.com 무료 플랜은 15분 무활동 시 슬립 모드로 전환됩니다.
첫 접속 시 30~60초 로딩 지연이 발생합니다.

**현재 완화책**

UptimeRobot에서 14분 간격으로 `/api/status` 핑 → 슬립 방지.

**완전 해결** (유료 전환 시)

Render.com Starter 플랜($7/월) 또는 Standard 플랜으로 업그레이드하면 슬립 없음.

---

### 3. 메모리 누수 (rateLimits 미정리)

**심각도**: 중 | **가능성**: 중간 (장시간 운영 시)

`rateLimits` Map이 소켓 해제 이벤트 없이 소켓이 사라질 경우 잔류할 수 있습니다.
Render.com 무료 플랜은 512MB 메모리 제한.

**권장 해결책** (v1.1 예정)

```js
setInterval(() => {
  const now = Date.now();
  for (const [key, times] of rateLimits) {
    const recent = times.filter(t => now - t < 60 * 1000);
    if (recent.length === 0) rateLimits.delete(key);
    else rateLimits.set(key, recent);
  }
}, 60 * 60 * 1000);  // 1시간마다 정리
```

### 4. 브라우저 자동재생 정책으로 사운드 무음

**심각도**: 하 | **가능성**: 높음

Chrome/Firefox 는 사용자 인터랙션 전 오디오 재생을 차단합니다.

**현재 완화책**

`sound.js` 에서 첫 `Sound.play()` 호출 시 lazy AudioContext 생성.
첫 수를 두거나 버튼을 클릭하면 자동으로 활성화됩니다.

---

### 5. iOS 긴 터치 OS 컨텍스트 메뉴 간섭

**심각도**: 하 | **가능성**: 중간

iOS Safari 에서 기물을 길게 누르면 OS 컨텍스트 메뉴가 열릴 수 있습니다.

**해결책** (v1.1 예정, CSS 2줄 추가):

```css
/* game.css — 각 보드 컨테이너에 추가 */
-webkit-touch-callout: none;
-webkit-user-select: none;
```

### 6. 채팅 내역 재접속 후 순서

**심각도**: 하 | **가능성**: 낮음

서버에서 `chat[]` 배열을 도착 순서대로 전송합니다.
두 플레이어가 동시에 메시지를 보낼 때 미묘한 순서 차이가 생길 수 있으나 실 영향은 미미합니다.

## 해결 완료 항목

### ✅ 15. 기권/무승부 버튼 도배 (클라이언트)

**해결일**: 2026-03-28

**이전 문제**: 무승부 제안·기권 버튼을 빠르게 반복 클릭하면 서버에 이벤트가 연속 전송됨.

**해결**:
- **무승부 제안**: 클릭 간 5초 딜레이, 3회 초과 시 60초 비활성화
- **기권**: 모달 열기 3초 쿨다운, 확인 버튼 이중 전송 방지 플래그
- **인디언 포커 액션**: fold/call/raise 1.5초 debounce

---

### ✅ 16. game:resign / game:draw:offer 서버 무제한 수신

**해결일**: 2026-03-28

**이전 문제**: 서버에서 `game:resign`, `game:draw:offer` 이벤트에 rate limit 없어
소켓 조작으로 무한 전송 가능했음.

**해결**: `rateCheck()` 적용

```js
// game:resign — 분당 최대 3회
if (!rateCheck(socket.id, 'resign', 3, 60 * 1000)) return;

// game:draw:offer — 분당 최대 5회
if (!rateCheck(socket.id, 'drawOffer', 5, 60 * 1000)) return;
```
