## 상세 분석

---

### 1. 서버 재시작 시 진행 중 게임 소멸

**심각도**: 상 | **가능성**: 높음

모든 게임 상태가 서버 메모리(in-memory Map)에만 존재합니다.
서버가 재시작되면 진행 중인 모든 게임 데이터가 사라집니다.

**현재 완화책**

- `SIGINT`/`SIGTERM` graceful shutdown: 종료 전 클라이언트에 `server:shutdown` 알림
- `stop.bat` 이 PowerShell `Get-Content` + `ConvertTo-Json` 으로 안정적으로 동작

**권장 해결책** (필요 시)

```js
// gracefulShutdown() 내에 추가
const dump = {};
for (const [id, room] of rooms) {
  if (room.status === 'active') {
    dump[id] = { fen: room.fen, moves: room.moves, timers: room.timers };
  }
}
fs.writeFileSync('state-dump.json', JSON.stringify(dump));
```

### 2. Cloudflare 터널 URL 변경

**심각도**: 중 | **가능성**: 높음

`trycloudflare.com` 무료 터널은 실행할 때마다 무작위 URL이 생성됩니다.

**현재 완화책**

- `start-public.ps1` 이 URL을 자동 캡처하여 콘솔에 표시
- 관리자 페이지 (`/admin.html`) 에서 실시간 표시, 클릭으로 복사

**권장 해결책** (고정 URL 필요 시)

```
cloudflared tunnel create chess-online
```

### 3. 메모리 누수 (장시간 운영)

**심각도**: 중 | **가능성**: 중간

`rateLimits` Map이 소켓 해제 이벤트 없이 소켓이 사라질 경우 잔류할 수 있습니다.

**권장 해결책** (24시간 이상 운영 시)

```js
setInterval(() => {
  const now = Date.now();
  for (const [key, times] of rateLimits) {
    const recent = times.filter(t => now - t < 60 * 1000);
    if (recent.length === 0) rateLimits.delete(key);
    else rateLimits.set(key, recent);
  }
}, 60 * 60 * 1000);
```

### 4. 브라우저 자동재생 정책으로 사운드 무음

**심각도**: 하 | **가능성**: 높음

Chrome/Firefox 는 사용자 인터랙션 전 오디오 재생을 차단합니다.

**현재 완화책**

`sound.js` 에서 첫 `Sound.play()` 호출 시 lazy AudioContext 생성.
첫 수를 두거나 버튼을 클릭하면 자동으로 활성화됩니다.

### 5. 포트 충돌 / 좀비 프로세스 ✅ 개선됨

**현재 개선 사항**

- `stop.bat`: 키 읽기를 PowerShell `Get-Content` 으로 변경
- `.shutdown-key` 없을 때 자동으로 `taskkill /F /IM node.exe` 강제 종료
- `start.bat`: `.shutdown-key` 존재 시 이미 실행 중 안내 후 종료

### 6. chess.js API 버전 불일치 ✅ 해결됨 (2026-03-21)

**이전 문제**: `package.json` 에 `"chess.js": "^0.12.0"` 으로 `^` 가 있어
`npm update` 시 v1.x API로 변경되어 TypeError 발생 가능.

**해결**: `"chess.js": "0.12.0"` 으로 버전 고정 (`^` 제거).

### 10. PC 절전/화면보호기로 서버 일시 정지

**심각도**: 중 | **가능성**: 높음

Windows 절전 모드 진입 시 Node.js 프로세스가 일시 정지되어:
- `timer:tick` 발송 정지 → 클라이언트 타이머 멈춤
- Socket.io 연결 끊김 가능성

**권장 해결책**

```
Windows 설정 → 전원 → 컴퓨터가 절전 모드로 전환: 없음
```

### 11. 인디언 포커 양측 칩 동시 부족 오판정 ✅ 해결됨 (2026-03-21)

**이전 문제** (`server.js:1543-1548`):

```js
// 수정 전 (잘못됨)
const winner = room.chips.host < room.ante ? 'black' : 'white';
// 양측 모두 < ante 이면 항상 'black' 판정
```

**해결**:
```js
if (room.chips.host < room.ante && room.chips.guest < room.ante) {
  // 칩 많은 쪽 승리, 동일하면 무승부(null)
  if (room.chips.host > room.chips.guest) winner = 'white';
  else if (room.chips.guest > room.chips.host) winner = 'black';
  else winner = null;
} else {
  winner = room.chips.host < room.ante ? 'black' : 'white';
}
```

### 12. 체커 재접속 시 이동 불가 ✅ 해결됨 (2026-03-21)

**이전 문제**:

클라이언트 `getAllCheckersValidMovesClient()` 함수가 항상 `[]` 반환.
재접속 시 서버에서 `validMoves`를 보내지 않아 이동 가능 칸 표시 불가.

**해결**:

서버 `room:reconnect` 핸들러에서 `game:state` 페이로드에 `validMoves` 추가:
```js
validMoves: room.gameType === 'checkers' && room.board
  ? (room.mustJump
      ? getValidCheckersMovesForPiece(...)
      : getAllCheckersValidMoves(room.board, room.currentTurn))
  : null
```

### 13. 게임 선택 취소 시 UI 상태 미초기화 ✅ 해결됨 (2026-03-21)

**이전 문제** (`lobby.js:133-137`):

사목·인디언 포커 선택 후 "뒤로 가기" 클릭 시:
- `color-section` 이 숨김 상태로 남아 다음 게임 선택 시 색상 선택 불가
- `selectedMinutes`, `isCustomTime`, 시간 버튼 active 상태 미초기화

**해결**: `backToGameSelect()` 함수에서 모든 UI 상태 완전 초기화.

### 14. 모바일 터치 타겟 크기 미달 ✅ 해결됨 (2026-03-21)

**이전 문제**:

Apple/Google HIG 권장 최소 터치 타겟 44×44px 미달:
- `.btn-control` (게임 제어 버튼): ~30px
- `.ip-action-btn` (인디언 포커 배팅): ~30px
- `.othello-cell` (480px 이하): 38px
- `.ck-cell` (480px 이하): 40px

**해결**: `min-height: 44px` 추가, 오셀로/체커 셀 크기 조정.

### 15. 모바일 뷰포트 overflow (오목 360px) ✅ 해결됨 (2026-03-21)

**이전 문제**:

480px 이하 오목 셀 24px → 15×24=360px + 패딩 → 360px 기기에서 가로 스크롤 발생.

**해결**: `@media (max-width: 360px) { .omok-cell { width:20px; height:20px } }`

### 16. 태블릿 게임 카드 그리드 미최적화 ✅ 해결됨 (2026-03-21)

**이전 문제**:

481~768px (태블릿) 구간에서 3컬럼 → 카드 너비가 좁아 텍스트 잘림.

**해결**: `@media (max-width: 768px) and (min-width: 481px)` 에서 2컬럼으로 전환.
